Each card tells you exactly where to verify the outcome.

Part 1 — End-to-end scenarios (send to queue expense-requests)

For each scenario: copy the JSON → open expense-requests queue → Service Bus Explorer → Send messages → Content Type application/json → paste body → Send. The Logic App picks it up within 30 sec. Check the Run history and your inbox.

Part 2 — Topic pub/sub with filtered subscriptions (send to topic expense-outcomes)

These three messages demonstrate the filtered subscription pattern directly. Send each message to the expense-outcomes topic with the listed custom property. Then peek each subscription to show that each one only receives messages matching its filter.

Part 3 — Verification checklist

After sending all 5 queue messages:
  1. Open ExpenseApproval → Run history. Expect 5 Succeeded runs.
  2. Click each run and screenshot the green workflow graph.
  3. Open your Gmail inbox — verify:
    • Scenario 1 → Expense AUTO_APPROVED: $45
    • Scenario 2 → Expense approval required (manager email; Approve/Reject buttons may be absent due to cross-tenant actionable-message policy)
    • Scenario 3 → same as 2, plus escalation if no action taken
    • Scenario 4 → Expense ESCALATED (after 2-min timeout)
    • Scenario 5 → Expense REJECTED (invalid data) with error list
  4. After sending the 3 topic messages: peek each subscription in the Azure Portal.
    • expense-approved → 1 message
    • expense-rejected → 1 message
    • expense-escalated → 1 message
    • all-outcomes → 3 messages (no filter)
  5. Screenshot each subscription's Peek result for the final report.
